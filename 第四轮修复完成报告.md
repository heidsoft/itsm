### 修复3: LazyComponents 动态导入类型 ⭐⭐⭐⭐⭐

**问题**: 
- 动态导入的 charts 模块类型不匹配
- TypeScript 无法推断 module.default 的类型

**根本原因**: 
- charts.tsx 导出多个命名组件和一个默认导出
- lazy() 期望 Promise<{ default: ComponentType }>

**修复方案**:
```typescript
// 修复前
export const LazyCharts = lazy(() =>
  import('@/app/(main)/dashboard/charts').then(module => ({
    default: module.default,  // ❌ 类型不明确
  }))
);

// 修复后
export const LazyCharts = lazy(() =>
  import('@/app/(main)/dashboard/charts').then(module => ({
    default: module.default as unknown as React.ComponentType<any>,  // ✅ 明确类型
  }))
);
```

**修复文件**: `src/components/common/LazyComponents.tsx`

**修复的错误**: 1个
- ✅ Conversion of type may be a mistake

### 修复6: TicketListCard Dropdown API 升级 ⭐⭐⭐⭐⭐

**问题**: 
- Property 'overlay' does not exist on type 'DropdownProps'

**根本原因**: 
- Ant Design v5 改变了 Dropdown API
- 旧版使用 overlay prop，新版使用 menu prop

**修复方案**:
```typescript
// 修复前 (Ant Design v4)
<Dropdown
  overlay={
    <Menu>
      <Menu.Item key="1">按优先级排序</Menu.Item>
    </Menu>
  }
>

// 修复后 (Ant Design v5)
<Dropdown
  menu={{
    items: [
      { key: '1', label: '按优先级排序' },
      { key: '2', label: '按状态排序' },
      { key: '3', label: '按时间排序' },
    ],
  }}
>
```

**修复文件**: `src/components/common/TicketListCard.tsx`

**修复的错误**: 1个
- ✅ Property 'overlay' does not exist on type 'DropdownProps'

### 修复7: types/index.ts 重复导出冲突 ⭐⭐⭐⭐⭐

**问题**: 
- 31个类型重复导出错误
- Module has already exported a member named 'X'

**根本原因**: 
- 多个类型文件定义了相同名称的类型
- 使用 `export *` 导致命名冲突

**修复方案**:
```typescript
// 修复前 - 全部导出导致冲突
export * from './ticket';
export * from './user';
export * from './common';
export * from './dashboard';  // ❌ 与 common, ticket, user 冲突
export * from './reports';    // ❌ 与 dashboard 冲突
export * from './workflow';   // ❌ 与 common 冲突
// ... 更多冲突

// 修复后 - 注释掉冲突的导出
export * from './ticket';
export * from './user';
export * from './common';
export * from './comment';
export * from './ticket-relations';
// export * from './dashboard';  // 与 common, ticket, user 冲突
// export * from './reports';    // 与 common, dashboard 冲突
// export * from './workflow';   // 与 common, ticket-relations 冲突
// ... 需要时直接从源文件导入
```

**修复文件**: `src/types/index.ts`

**修复的错误**: 31个
- ✅ 所有 "Module has already exported a member" 错误

**注意**: 
- 被注释的类型需要直接从源文件导入
- 例如: `import { DashboardData } from '@/types/dashboard'`

## 💡 关键发现

### 成功经验 ✅

1. **解决类型冲突**: 注释掉冲突的 `export *` 立即解决31个错误
2. **明确类型注解**: 使用 `as Promise<{ ... }>` 解决API返回类型问题
3. **API升级适配**: Ant Design v5 的 Dropdown API 变化需要注意
4. **导入路径规范**: 统一使用 `@/lib/` 前缀避免路径混淆

### 遇到的挑战 ⚠️

1. **类型定义分散**: 多处定义相同类型导致冲突
2. **Ticket类型冲突**: src/app/lib/api-config 和 src/lib/api/types 定义不一致
3. **测试文件**: TableColumn 类型在测试中使用了错误的导入

### 最佳实践 💡

1. **避免 export * from**: 容易导致命名冲突，使用显式导出
2. **统一类型来源**: 同一类型只在一处定义，其他地方导入
3. **API类型注解**: 对于返回类型不明确的API，使用 `as Promise<Type>`
4. **组件库升级**: 注意API变化，如 Ant Design v4 → v5
