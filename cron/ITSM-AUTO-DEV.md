# ITSM 自动化开发 Cronjob

## ⚠️ 重要说明

**自动化开发有限制**，为避免自动提交有问题的代码：

- ✅ **可以做**: 文档更新、简单重构、配置修改、测试编写
- ⚠️ **需审查**: 业务逻辑修改、新功能开发、依赖变更
- ❌ **禁止**: 删除代码、核心逻辑修改、生产配置变更

---

## 📋 配置信息

| 配置项 | 值 |
|--------|-----|
| **名称** | `itsm-auto-dev` |
| **描述** | 根据 ITSM 任务列表自动完成开发任务并提交 GitHub |
| **执行频率** | 每天 10:00（可调整） |
| **时区** | Asia/Shanghai |
| **Agent** | `dev` (小开) |
| **会话模式** | `isolated`（独立会话） |
| **通知方式** | `announce`（推送到最后聊天） |
| **超时** | 300 秒（5 分钟） |

## 🕐 Cron 表达式

```
0 10 * * *  # 每天早上 10 点执行
```

---

## 🔄 工作流程

```
1. 读取任务列表
   ↓
2. 选择合适任务（P1/P2，小任务）
   ↓
3. 分析任务需求
   ↓
4. 编写/修改代码
   ↓
5. 本地验证
   ↓
6. 生成变更报告
   ↓
7. 等待审查 或 Push 到 feature 分支
   ↓
8. 输出开发报告
```

## 📝 任务选择策略

### 自动执行（无需审查）
- [x] 文档更新（README、注释等）
- [x] 代码格式化
- [x] 简单重构（变量命名、函数提取）
- [x] 添加单元测试
- [x] 配置文件更新

### 需审查后执行
- [ ] 新功能开发
- [ ] 业务逻辑修改
- [ ] API 变更
- [ ] 数据库 Schema 修改

### 禁止自动执行
- [x] 删除代码/文件
- [x] 核心业务逻辑修改
- [x] 生产环境配置
- [x] 破坏性变更

## 🔧 管理命令

```bash
# 查看状态
openclaw cron list

# 查看执行历史
openclaw cron runs --name "itsm-auto-dev"

# 手动触发一次
openclaw cron run --name "itsm-auto-dev"

# 修改频率（如改为每周一 10 点）
openclaw cron edit itsm-auto-dev --cron "0 10 * * 1"

# 禁用/启用
openclaw cron disable itsm-auto-dev
openclaw cron enable itsm-auto-dev
```

## 💡 最佳实践

### 第一天：观察模式
```bash
# 禁用自动执行，只生成报告
openclaw cron disable itsm-auto-dev

# 手动运行一次查看效果
openclaw cron run --name "itsm-auto-dev"
```

### 第一周：审查模式
- 每天查看生成的报告
- 审查 feature 分支的变更
- 确认无误后手动合并

### 稳定后：自动模式
- 信任自动化流程
- 定期审查代码质量
- 调整任务选择策略

## ⚠️ 安全机制

### 1. 变更限制
- 单次最多修改 5 个文件
- 单文件最多修改 100 行
- 不删除任何文件

### 2. 验证机制
- 必须通过代码格式检查
- 必须通过类型检查（如有）
- 不能引入编译错误

### 3. 审查机制
- 复杂任务创建 feature 分支
- 生成详细的变更报告
- 等待人工确认

### 4. 回滚机制
- 所有变更都有 git 记录
- 可随时回滚到任意版本
- 保留完整的提交历史

## 🎯 适用场景

### ✅ 适合自动化
- 文档维护和更新
- 代码格式统一
- 简单 Bug 修复
- 测试用例补充
- 日志/注释完善

### ⚠️ 需要审查
- 新功能开发
- 接口变更
- 性能优化
- 重构改进

### ❌ 不适合自动化
- 架构设计
- 核心算法
- 安全相关
- 用户体验决策
