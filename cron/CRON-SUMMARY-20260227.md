## 🔧 管理命令

### 查看所有 Cronjobs
```bash
openclaw cron list
```

### 查看执行历史
```bash
openclaw cron runs
```

### 手动触发
```bash
# 触发指定 ID 的任务
openclaw cron run <job-id>

# 例如：手动触发代码质量检查
openclaw cron run ed3d1a01-d50a-4d85-ae9c-7df72fe3bab2
```

### 修改频率
```bash
# 例如：改为每 2 小时检查一次代码质量
openclaw cron edit ed3d1a01-d50a-4d85-ae9c-7df72fe3bab2 --cron "0 */2 * * *"
```

### 禁用/启用
```bash
# 禁用
openclaw cron disable <job-id>

# 启用
openclaw cron enable <job-id>

# 删除
openclaw cron rm <job-id>
```

## ⚠️ 注意事项

1. **通知管理**: 6 个 Cronjob 会产生较多通知，建议：
   - 重要通知：立即推送
   - 常规报告：汇总推送
   - 周报：单独推送

2. **审查频率**: 
   - feature 分支：每天审查一次
   - 安全报告：每天查看
   - 质量报告：每周查看趋势

3. **资源消耗**:
   - API 调用：GitHub API 有速率限制
   - 计算资源：复杂分析可能耗时
   - 存储：历史报告会占用空间

4. **灵活调整**:
   - 根据实际效果随时调整
   - 节假日可暂停自动开发
   - 发布期间可增加监控频率
