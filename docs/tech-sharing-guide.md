# 技术分享机制

> 适用于 1-2 人小团队的轻量级技术分享体系

## 1. 分享形式

### 1.1 定期分享（每周/双周）

| 类型 | 时长 | 频率 | 形式 |
|------|------|------|------|
| 技术小贴士 | 10-15 分钟 | 每周 | 站会后 |
| 深度分享 | 30-45 分钟 | 双周 | 专门会议 |

### 1.2 分享内容来源

```
┌─────────────────────────────────────────────────────────────┐
│                      分享主题来源                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │

│  │  本周学习   │    │  项目问题   │    │  新技术   │   │
│  │  (30%)     │    │  (40%)     │    │  (30%)    │   │
│  └─────────────┘    └─────────────┘    └─────────────┘   │
│                                                             │
│  • 读书/文章心得      • 线上问题复盘      • 新框架调研    │
│  • 培训课程总结      • Bug 根因分析      • 工具推荐      │

```

## 2. 分享主题规划

### 2.1 主题池（按优先级排序）

#### P0 - 立即需要（当前项目相关）

| 主题 | 类型 | 预计时长 | 状态 |
|------|------|-----------|------|
| 项目架构回顾 | 深度 | 45 分钟 | 待安排 |
| API 设计最佳实践 | 深度 | 30 分钟 | 待安排 |
| 测试驱动开发 TDD 入门 | 深度 | 45 分钟 | 待安排 |
| 错误处理与日志规范 | 小贴士 | 15 分钟 | 待安排 |

#### P1 - 近期需要（技能提升）

| 主题 | 类型 | 预计时长 | 状态 |
|------|------|-----------|------|

| 前端性能优化 | 深度 | 30 分钟 | 待安排 |
| 数据库索引优化 | 深度 | 30 分钟 | 待安排 |
| Docker & K8s 基础 | 深度 | 45 分钟 | 待安排 |

#### P2 - 长期储备（技术视野）

| 主题 | 类型 | 预计时长 | 状态 |

## 3. 分享流程

### 3.1 标准流程

```
┌──────────────────────────────────────────────────────────────────┐
│                        分享准备流程                               │
├──────────────────────────────────────────────────────────────────┤
│                                                                   │
│  Day -7          Day -3           Day -1          分享日       │
│    │                │                │               │           │
│    ▼                ▼                ▼               ▼           │
│ ┌──────┐        ┌──────┐        ┌──────┐       ┌──────┐      │
│ │确定  │   ──▶ │准备  │   ──▶ │预演  │  ──▶ │正式  │      │
│ │主题  │        │材料  │        │(可选) │       │分享  │      │
│ └──────┘        └──────┘        └──────┘       └──────┘      │
│                                                                   │
│  • 查阅文档        • PPT/演示          • 计时       • 互动     │
│  • 收集资料        • 代码示例        • 录音       • Q&A      │
│                   • 演示Demo                                     │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

### 3.2 分享结构模板

```markdown
# [主题名称]

## 背景 (2分钟)
- 为什么分享这个？
- 解决什么问题？

## 核心内容 (N分钟)
- 关键概念
- 代码示例
- 演示 Demo

## 实践要点 (2分钟)
- 如何应用到项目？
- 注意事项

## Q&A (5分钟)
- 开放讨论

## 参考文献
- 文档/文章/视频链接
```

### 3.3 分享材料要求

| 类型 | 要求 |
|------|------|
| PPT | 简洁为主，避免大段文字，每页 < 6 行 |
| 代码 | 必须可运行，提前验证 |
| 演示 | 提前演练，预估时间 |
| 文档 | 分享后归档到知识库 |

## 4. 知识沉淀

### 4.1 知识库结构

```
docs/
├── tech-sharing/           # 技术分享
│   ├── 2024/              # 按年份
│   │   ├── 01-golang-concurrency.md
│   │   ├── 02-react-performance.md
│   │   └── ...
│   └── template.md         # 分享模板
│
├── architecture/           # 架构决策
│   ├── adr-001-use-ent-orm.md
│   └── ...
│
├── best-practices/        # 最佳实践
│   ├── go-error-handling.md
│   └── ...
│
└── learning/             # 学习笔记
    ├── books/
    └── courses/
```

### 4.2 知识沉淀清单

每次分享后必须完成：

- [ ] 分享 PPT/演示材料 (PDF/HTML)
- [ ] 代码示例 (可运行)
- [ ] 关键要点笔记 (Markdown)
- [ ] 参考资料链接
- [ ] 录音/录像（可选）

### 4.3 知识复用

```
分享产出物              复用场景
─────────────────────────────────────────────

PPT/演示材料     ──▶  团队新人 onboarding
                        客户技术交流
                        招聘面试展示

代码示例        ──▶  项目模板
                        最佳实践参考
                        面试答题素材

技术笔记        ──▶  知识库搜索
                        培训材料
                        Wiki 文档

```

## 6. 实施计划

### 6.1 启动阶段（第 1-2 周）

- [ ] 确定首次分享主题和日期
- [ ] 准备分享模板和知识库结构
- [ ] 设立分享日历邀请

### 6.2 养成阶段（第 3-8 周）

- [ ] 每周一次技术小贴士（站会后）
- [ ] 双周一次深度分享
- [ ] 每次分享后归档知识库

### 6.3 优化阶段（持续）

- [ ] 根据反馈调整分享形式
- [ ] 建立主题池更新机制
- [ ] 探索更多复用方式

## 7. 附录

   - 技术选型理由
   - 核心模块介绍

2. **Git 工作流与 Code Review 实践** (30 分钟)
   - 分支策略
   - PR 规范
   - 审查要点

3. **测试实践分享** (30 分钟)
   - 单元测试写法
   - 覆盖率目标
