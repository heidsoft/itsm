# 个人中心模块测试用例

## 1. 功能概述

个人中心模块是ITSM系统中用户管理自身账户信息的核心功能模块，提供用户查看、编辑个人资料、修改密码、管理偏好设置和安全设置等功能。该模块与其他业务模块（如工单、通知等）有数据关联，需要确保数据一致性和安全性。

### 1.1 功能边界

| 功能模块 | 描述 |
|---------|------|
| 个人信息查看 | 显示用户名、邮箱、角色、部门、头像、联系方式、创建时间、最后登录时间、账号状态 |
| 个人信息编辑 | 修改昵称、邮箱（需验证）、手机号、部门（需权限）、头像上传 |
| 密码修改 | 旧密码验证、新密码设置、确认密码一致性、密码强度检查、修改后其他设备登出 |
| 偏好设置 | 语言切换、主题切换、时区设置、日期格式设置、通知偏好 |
| 安全设置 | 登录日志查看、操作日志查看、活跃会话管理、第三方应用授权、API Keys管理 |

### 1.2 用户角色与权限

| 角色 | 权限说明 |
|------|---------|
| 普通用户 | 查看和编辑本人信息、修改密码、管理偏好设置、查看登录日志 |
| 管理员 (admin) | 普通用户权限 + 查看所有用户信息、修改部门字段 |
| 超级管理员 (super_admin) | 管理员权限 + 强制重置密码、查看所有操作日志 |

## 4. 风险与注意事项

### 4.1 高风险场景

| 场景 | 风险等级 | 说明 |
|------|---------|------|
| 密码修改功能 | 高 | 涉及账户安全，需严格验证 |
| 邮箱修改 | 高 | 需要验证防止账户被盗 |
| 两步验证 | 高 | 影响登录流程，务必确保可恢复 |
| 活跃会话管理 | 中 | 误操作可能导致用户意外登出 |

### 4.2 测试环境要求

- 测试环境应与生产环境隔离
- 密码修改测试后需要重置测试账号密码
- 第三方授权测试需要准备测试用的第三方应用
- 网络异常场景需要使用工具模拟网络中断

### 4.3 回归测试建议

每次发布前应重点回归以下用例：
- TC-PROFILE-INFO-008（未登录访问重定向）
