## 附录

### 优先级定义
- **P0**: 核心功能，必须测试通过
- **P1**: 重要功能，建议测试通过
- **P2**: 辅助功能，根据时间安排测试

### 状态流转图
```
open -> in_progress -> resolved -> closed
                    -> (reopen) -> open
```

### API端点参考
- `POST /api/v1/problems` - 创建问题
- `GET /api/v1/problems` - 获取问题列表
- `GET /api/v1/problems/:id` - 获取问题详情
- `PUT /api/v1/problems/:id` - 更新问题
- `POST /api/v1/problems/:id/investigate` - 问题调查
- `POST /api/v1/problems/:id/root-cause` - 记录根本原因
- `POST /api/v1/problems/:id/solution` - 提供解决方案
- `POST /api/v1/problems/:id/close` - 关闭问题
