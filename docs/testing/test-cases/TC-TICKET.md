# 工单模块测试用例

## 功能概述

工单模块是ITSM系统的核心模块之一，负责IT服务请求的创建、处理、跟踪和关闭的全生命周期管理。本文档涵盖工单的创建、查询、编辑、分配、升级、解决/关闭、自动化规则、分配规则以及导入导出等功能的测试用例。

## 1. 工单创建

### TC-TICKET-CREATE-001
- **测试标题**: 正常创建工单 - 填写所有必填字段
- **优先级**: P0
- **前置条件**: 用户已登录系统，拥有工单创建权限
- **测试步骤**:
  1. 点击"新建工单"按钮
  2. 选择工单类型（请求/事件/问题）
  3. 填写标题（长度50字符）
  4. 填写详细描述（长度500字符）
  5. 选择优先级（低/中/高/紧急）
  6. 选择工单分类（一级分类和二级分类）
  7. 选择请求类型（硬件/软件/网络/其他）
  8. 选择影响范围（个人/部门/全局）
  9. 设置期望完成时间
  10. 上传一个不超过10MB的附件
  11. 点击"提交"按钮
- **预期结果**: 工单创建成功，系统生成唯一工单编号（如TKT-2026-00001），页面跳转到工单详情页，显示创建成功提示

## 测试用例执行说明

### 执行优先级
- **P0（阻塞级）**: 必须执行，阻塞后续测试
- **P1（高优先级）**: 必须执行，发现问题需立即修复
- **P2（中等优先级）**: 建议执行，发现问题可安排修复

### 测试环境要求
- 测试环境应与生产环境配置一致
- 测试数据应包含正常数据和边界数据
- 每个测试用例应独立执行，避免相互依赖

### 缺陷管理
- 发现缺陷时，记录缺陷编号、严重程度、复现步骤
- P0/P1缺陷需在当前迭代修复
- P2缺陷可根据项目计划安排修复

### 测试完成标准
- 所有P0测试用例通过
- P1测试用例通过率 > 95%
- P2测试用例通过率 > 85%
