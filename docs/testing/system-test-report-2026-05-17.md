### 1.1 浏览器矩阵（目标覆盖≥95%市场份额）

## 3. 核心功能验证要点

### 3.1 跨角色权限流转

- 角色覆盖：普通提报人（end_user）、运维工程师（agent/engineer）、审批经理（approver/manager）、系统管理员（admin）
- 验证点：菜单可见性、页面访问控制（403/跳转）、操作级权限（创建/编辑/审批/关闭）

### 3.2 工单/流程全生命周期

- 工单：创建 → 分派 → 处理 → 审批 → 关闭（含升级、评论、附件、通知）
- 变更/发布：创建 → 审批 → 执行 → 验证 → 关闭（含回滚）
- CMDB：CI创建/编辑 → 关系维护 → 影响分析（与事件/问题/变更的关联准确性）

### 3.3 通知与消息

- 站内信：通知中心展示、跳转链路、重复触发控制
- 邮件：触发条件、模板变量正确性、收件人范围正确性

| 类型 | 命令 | 说明 |
