## 四、数据模型审查

### 4.1 核心实体关系

```
Ticket (工单)
├── 关联 Template (模板)
├── 关联 Category (分类)
├── 关联 Department (部门)
├── 关联 Tags (标签)
├── 关联 SLA Definition (SLA定义)
├── 关联 Comments (评论)
├── 关联 Attachments (附件)
├── 关联 Problems (问题)
├── 关联 Configuration Items (配置项)
└── 关联 Approval Records (审批记录)

SLA Definition (SLA定义)
├── 关联 Tickets (工单)
├── 关联 Violations (违规记录)
├── 关联 Metrics (指标)
└── 关联 Alert Rules (预警规则)

Change (变更)
├── 关联 Approval Workflows (审批流程)
└── 关联 Tickets (工单)
```

### 4.2 Schema评估

**优点**:

- 字段命名统一使用snake_case
- 包含审计字段(created_at, updated_at)
- 支持软删除(deleted_at)
- 有版本控制(version字段)
- 索引设计合理

**改进建议**:

- 建议添加 last_response_at 字段追踪首次响应
- 建议添加 source 字段标识工单来源(自助门户/邮件/电话)
- 建议添加 urgency 和 impact 字段完善事件分级
