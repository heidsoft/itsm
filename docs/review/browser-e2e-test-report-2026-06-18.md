### 1. 登录页面 (/login) — 发现 Bug（已修复）

**截图**: login-page.png

**页面质量**: ★★★★★

**发现的问题**:
> **CSRF Token 端点路径不匹配**
> - 前端调用: `GET /api/v1/auth/csrf` → 返回 **404**
> - 后端实际: `GET /api/v1/csrf-token` → 返回 **200**
> - **根因**: `itsm-frontend/src/lib/api/auth-api.ts` 第59行路径错误
> - **修复**: 已将 `CSRF: '/api/v1/auth/csrf'` 改为 `CSRF: '/api/v1/csrf-token'`

**BUG-001 详细说明**:
- **文件**: `itsm-frontend/src/lib/api/auth-api.ts`

## 四、基础设施问题

| # | 问题 | 影响 | 建议 |
|---|------|------|------|
| INF-01 | Redis 未配置密码（NOAUTH required） | 序列号服务降级为数据库模式；限流降级为内存模式 | docker-compose 中添加 `REDIS_PASSWORD` 并在 .env 配置 |
| INF-02 | pgvector 扩展未安装 | RAG 向量检索降级为关键字搜索 | 在 PostgreSQL 容器中安装 pgvector 扩展 |
| INF-03 | Docker 前端容器无代码挂载 volume | 本地修改不生效，需重启 dev server 或重新构建镜像 | 开发阶段使用 npm run dev，生产阶段配置 volume mount |
