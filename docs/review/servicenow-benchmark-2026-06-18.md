## 四、后端架构深度评估

### 架构评分：4.1 / 5.0（商用就绪）

**优势：**
- ✅ **Go/Gin + Ent ORM**：类型安全，代码生成减少错误，高并发性能好
- ✅ **安全加固完成**：SQL 注入、XSS、CSRF、速率限制、安全响应头全覆盖
- ✅ **多租户隔离**：每请求 TenantMiddleware + 数据层过滤，支持 MSP 场景
- ✅ **RBAC 三层模型**：role → permission → ACL expression，动态细粒度授权
- ✅ **AI 集成架构**：LLM Gateway 多提供商、pgvector RAG、triage_service 分诊
- ✅ **可观测性**：Prometheus metrics + Grafana + SLA 告警规则配置就绪
- ✅ **事务保护**：Ent ORM 事务 + 乐观锁（process_instance.version）

**待改进：**
- ⚠️ **service/ 层过重**：156 个文件，ticket 相关拆分过细（ticket_core/ticket_lifecycle/ticket_service_v2 等），建议合并
- ⚠️ **BPMN 引擎局限**：自研引擎在复杂流程（多实例、子流程、消息边界事件）方面有局限，长期建议考虑 Flowable/Camunda 适配层
- ⚠️ **前端与后端 DTO 对齐**：部分前端组件仍有硬编码字段，需对齐 API 响应 camelCase 规范

## 五、开源 ServiceNow 路线图

### 竞争定位

```
ServiceNow          → 企业级闭源，价格高，定制难
OTRS / iTop         → 开源但 UI 过时，AI 能力弱
Jira Service Mgmt   → 偏项目管理，ITIL 不完整
osTicket            → 轻量但功能单薄
本项目（目标）      → 开源 + AI-Native + 低代码 + 现代 UI
```

### 版本规划

#### v1.0 — 商用基础 ✅（当前已完成）
- ITIL 四大流程（事件/问题/变更/发布）
- 多租户 RBAC + 安全加固
- SLA 监控 + 告警
- BPMN 工作流引擎
- AI 工单分诊 + RAG 知识检索
- CMDB 基础 + CI 拓扑

#### v1.1 — UX 强化（1-2 个月）
**核心目标：达到 ServiceNow 操作员端的操作流畅度**
- [ ] 工单 Kanban 看板视图（拖拽换状态）
- [ ] 移动端响应式（关键流程页面）
- [ ] 深色模式
- [ ] 批量操作（分配/关闭/导出）
- [ ] 搜索体验优化（全局搜索结果分类展示）
- [ ] 通知中心 UX 重构

#### v1.2 — 低代码平台（2-3 个月）
**核心目标：超越 iTop/OTRS，向 ServiceNow 低代码能力靠齐**
- [ ] **表单设计器**（拖拽字段：文本/选择/日期/文件/关联）
- [ ] **BPMN 可视化设计器**（引入 bpmn-js，替换当前基础版）
- [ ] 自定义字段（工单/CMDB CI 类型扩展字段）
- [ ] 规则引擎可视化配置（自动化规则/分配规则可视化）

#### v1.3 — 自助门户（3-4 个月）
**核心目标：ServiceNow Service Portal 对标**
- [ ] 独立 ITSM Portal（面向普通用户）
- [ ] 用户自定义仪表盘（Widget 拖拽）
- [ ] 我的工单跟踪（简化视图）
- [ ] 知识库自助搜索门户
- [ ] 通知偏好中心

#### v1.4 — 企业集成（4-5 个月）
- [ ] SSO / LDAP / SAML 完整支持
- [ ] OpenAPI 文档（Swagger UI 集成）
- [ ] 多语言（中/英，next-intl + Go i18n）
- [ ] 飞书/钉钉/企业微信通知完整对接
- [ ] CMDB 自动发现（K8s/云 API 同步）
- [ ] 报表导出（PDF/Excel）

#### v2.0 — AI-Native ITSM（6 个月+）
**核心目标：构建超越 ServiceNow 标准版的 AI 差异化能力**
- [ ] AI Agent 自动分诊闭环（无人工干预的故障处理）
- [ ] 自然语言工单（语音/文字 → 标准工单）
- [ ] 预测性 SLA（基于历史数据预测违约风险）
- [ ] 智能根因分析（CMDB + 日志 + 告警关联分析）
- [ ] AI 变更风险评估
- [ ] Plugin 市场（第三方集成生态）

## 六、前端体验专项建议

### 当前问题

1. **缺少看板视图**：工单列表只有表格，缺少 Kanban/看板视图，处理效率低
2. **表单体验弱**：服务目录申请表单为静态表单，无法按服务类型动态渲染
3. **移动端适配差**：多数页面在移动端压缩严重，关键操作难以完成
4. **Dashboard 数据孤立**：各模块 Dashboard 独立，缺少统一的 KPI 视图
5. **搜索不智能**：GlobalSearch 已有但结果展示需改善，无高亮/权重排序
6. **UI 一致性**：37 个模块 UI 风格略有差异，组件库使用不统一

### 优先修复建议

```
第一优先级（影响日常使用）：
  → 工单看板 Kanban（2周）
  → 关键页面移动端适配（2周）
  → 统一 Loading/Empty/Error 状态组件（1周）

第二优先级（影响核心功能）：
  → 表单设计器 MVP（4周）
  → 自定义仪表盘（4周）

第三优先级（体验优化）：
  → 深色模式（1周）
  → 批量操作（1周）
  → 搜索结果优化（2周）
```
