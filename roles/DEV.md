## 🔧 开发规范

### 代码规范

#### Go 代码规范

```go
// ✅ 推荐：清晰的函数命名
func GetUserByID(id int64) (*User, error) {
    // 实现
}

// ❌ 避免：模糊的命名
func get(id int) (*xxx, error) {
    // 实现
}

// ✅ 推荐：错误处理
if err != nil {
    return fmt.Errorf("failed to get user: %w", err)
}

// ✅ 推荐：注释规范
// GetUserByID retrieves user by ID from database
// Returns error if user not found
func GetUserByID(id int64) (*User, error) {
    // 实现
}
```

#### TypeScript 代码规范

```typescript
// ✅ 推荐：明确的类型定义
interface User {
  id: number;
  name: string;
  email: string;
}

// ❌ 避免：使用 any
interface User {
  id: any;
  name: any;
}

// ✅ 推荐：函数类型
const getUser = async (id: number): Promise<User> => {
  // 实现
};

// ✅ 推荐：错误处理
try {
  const user = await getUser(1);
} catch (error) {
  console.error('Failed to get user:', error);
}
```

### Git 工作流

#### 分支管理

```
main (生产环境)
  ↑
develop (开发环境)
  ↑
feature/xxx (功能分支)
  ↑
bugfix/xxx (bug 修复)
  ↑
hotfix/xxx (紧急修复)
```

#### 提交信息规范

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type 类型**:
- `feat`: 新功能
- `fix`: bug 修复
- `docs`: 文档更新
- `style`: 代码格式
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

**示例**:
```
feat(ticket): add ticket rating feature

- Add rating component
- Add rating API
- Add rating database schema

Closes #123
```

### 代码审查 (Code Review)

#### 审查清单

**代码质量**:
- [ ] 代码是否清晰易懂
- [ ] 是否有重复代码
- [ ] 是否遵循编码规范
- [ ] 是否有适当的注释

**功能实现**:
- [ ] 是否实现所有需求
- [ ] 是否有边界条件处理
- [ ] 是否有错误处理
- [ ] 是否有日志记录

**性能优化**:
- [ ] 是否有性能问题
- [ ] 是否有内存泄漏
- [ ] 是否有数据库优化
- [ ] 是否有缓存机制

**安全性**:
- [ ] 是否有 SQL 注入风险
- [ ] 是否有 XSS 风险
- [ ] 是否有 CSRF 风险
- [ ] 是否有权限验证

#### 审查流程

```
开发者提交 MR → 自动 CI 检查 → 同事审查 → 修改完善 → 合并代码
     ↓            ↓            ↓          ↓          ↓
  GitLab       GitHub       1-2 人      按需        main
```
