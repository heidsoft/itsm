## 📖 技能说明

这个 skill 提供了一套系统性的方法来诊断和修复 CI/CD 构建失败问题，优化构建流程，提升构建成功率和性能。

### 适用场景

- ✅ GitHub Actions 构建持续失败
- ✅ npm/Go 依赖管理问题
- ✅ 代码质量检查失败
- ✅ YAML 配置错误
- ✅ 构建性能需要优化

### 不适用场景

- ❌ 业务逻辑错误
- ❌ 功能需求开发
- ❌ 架构设计问题

## 🔧 常见问题

### Q1: 构建持续失败怎么办？

**A**: 按照 Phase 1 诊断问题，查看构建日志，识别错误类型，然后选择相应的修复方案。

### Q2: 依赖冲突如何解决？

**A**: 参考 Phase 2，使用 `--legacy-peer-deps` 或升级冲突的依赖版本。

### Q3: YAML 语法错误如何排查？

**A**: 参考 Phase 3.1，检查引号、缩进、变量拼接等常见错误。

### Q4: 代码格式化问题如何处理？

**A**: 参考 Phase 4.1，使用 gofumpt（Go）或 Prettier（TypeScript）格式化代码。
