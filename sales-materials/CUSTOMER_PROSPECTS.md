#### 需求挖掘（3-5 分钟）

**问题清单**:

1. **现状了解**
   - "贵公司目前有多少 IT 人员？"
   - "每天大概处理多少工单？"
   - "目前使用什么工具管理工单？"

2. **痛点挖掘**
   - "在工单处理过程中，遇到的最大挑战是什么？"
   - "客户/员工对 IT 服务的满意度如何？"
   - "有没有遇到过工单响应不及时被投诉的情况？"

3. **需求确认**
   - "如果有一个系统可以自动分派工单、监控 SLA，对您有帮助吗？"
   - "您希望系统解决哪些具体问题？"
   - "您理想中的 IT 运维管理是什么样的？"

4. **预算了解**
   - "贵公司今年在 IT 运维管理方面有计划吗？"
   - "大概的预算范围是多少？"
   - "决策流程是怎样的？"

#### 异议处理

**异议 1: "我们已经有系统了"**
```
理解，很多客户在接触我们之前都有在使用其他系统。

他们选择我们的原因主要是：
1. 我们的 AI 智能分派可以节省 90% 的人工分派时间
2. 我们的 SLA 管理可以确保工单 100% 及时处理
3. 我们的知识库可以让新人快速上手

您看要不要安排一个演示，
您对比一下看看我们有什么不同的地方？
```

**异议 2: "价格太贵了"**
```
理解您的考虑。

其实从长期来看，我们的系统可以帮助您：
- 减少 30% 的人力成本
- 降低 50% 的系统故障时间
- 提升 40% 的客户满意度

算下来，投资回报率通常在 6 个月内就能收回。

而且我们可以根据您的实际情况调整套餐方案，
您看要不我们先安排一个演示，
看看系统是否真的能帮您解决问题？
```

**异议 3: "需要考虑一下"**
```
完全理解，这是一个重要决策。

为了帮助您更好地评估，我可以：
1. 安排一个 30 天的免费试用
2. 提供同行业的客户案例
3. 安排一次技术答疑会议

您看哪个对您更有帮助？

另外，我们这个月有一个限时优惠，
前 100 名客户可以免费获得实施服务（价值¥30 万），
您看要不要先锁定这个优惠？
```
