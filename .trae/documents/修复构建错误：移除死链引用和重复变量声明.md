# 构建错误修复计划

## 问题汇总
1.  **Module not found**: `./src/app/(main)/my-requests/page.tsx` 引用了已删除的 `mock-data.ts` 文件。
2.  **Duplicate identifier**: `./src/app/(main)/tickets/analytics/page.tsx` 中 `analyticsData` 变量被重复声明。

## 修复计划

### 1. 修复 My Requests 页面
- **文件**: `src/app/(main)/my-requests/page.tsx`
- **问题**: 第 50 行 `import { mockRequestsData } from '@/app/lib/mock-data';` 引用了已删除的文件。且 `loadMockData` 函数依赖于该导入。
- **操作**:
    - 删除 `mock-data` 的导入。
    - 删除 `loadMockData` 函数及其所有引用。
    - 修改 `fetchRequests` 函数：在 `catch` 块中，不再调用 `loadMockData`，而是直接设置空数组 `setRequests([])` 并打印错误日志。

### 2. 修复 Ticket Analytics 页面
- **文件**: `src/app/(main)/tickets/analytics/page.tsx`
- **问题**: 代码中存在 `const [analyticsData, setAnalyticsData] = useState...` 的重复声明，或者变量名冲突。根据之前的修改记录，可能是因为在重构时没有正确清理旧的 `useState` 声明。
- **操作**:
    - 检查并移除重复的 `const [analyticsData, setAnalyticsData]` 声明。确保只保留一个状态定义。

## 预期结果
- 执行修复后，`npm run build` 应该能顺利通过编译。
- 所有页面将不再依赖已删除的 Mock 文件。
- 变量重复声明错误将被解决。
